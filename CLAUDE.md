# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm install          # install deps (also run after pulling changes to package-lock.json)
npm run setup        # copy Readability.js → lib/ and regenerate icons/ (run after npm install)
npm run lint         # oxlint
npm run format       # oxfmt (write)
npm run check        # lint + format check (no writes)
```

There is no build step and no test suite — reload the extension in `chrome://extensions` to test changes.

## Architecture

This is a Chrome Manifest V3 extension. There is no bundler; every `.js` file is loaded directly by the browser.

**Three execution contexts that cannot share memory:**

| Context | File(s) | Can reach |
|---|---|---|
| Service worker | `background.js` | `chrome.*` APIs only |
| Content script | `lib/Readability.js`, `content.js` | Page DOM + `chrome.runtime` |
| Extension pages | `sidepanel.js`, `settings.js` | `chrome.*` APIs + `fetch` to localhost |

**Why `background.js` exists:** Content scripts cannot message extension pages directly. `background.js` acts purely as a relay — it re-broadcasts any content-script message to `chrome.runtime` with the originating `tabId` attached, so `sidepanel.js` can filter by tab.

**All Ollama `fetch` calls happen in `sidepanel.js`** — extension pages can reach `localhost`; content scripts cannot.

### Message protocol

```
sidepanel.js  ──chrome.tabs.sendMessage(tabId, {type, ...})──►  content.js
content.js    ──chrome.runtime.sendMessage({type, tabId, ...})──►  background.js (relay)
background.js ──chrome.runtime.sendMessage({...relayed, tabId})──►  sidepanel.js
```

Message types (content script handles): `ENTER_PICKER_MODE`, `EXIT_PICKER_MODE`, `EXTRACT_FULL_PAGE`, `EXTRACT_ELEMENT`, `GET_PAGE_INFO`.  
Message types (side panel handles): `ELEMENT_SELECTED`, `PICKER_CANCELLED`.

### Storage schema (`chrome.storage.local`)

```jsonc
{
  "endpoint": "http://localhost:11434",
  "defaultModel": "llama3.2",
  "models": ["llama3.2", ...],        // cached from Ollama /api/tags
  "pageSelections": {
    "https://example.com/page": { "selector": "article", "label": "article", "savedAt": 1716300000000 }
  },
  "pageModels": {
    "https://docs.example.com": "mistral"
  },
  "summaryDetail": "standard"         // "brief" | "standard" | "detailed"
}
```

URLs stored as keys are normalised (hash stripped, trailing slash removed) via `normalizeUrl()` in `sidepanel.js`.

### `sidepanel.js` state model

The side panel persists per-tab UI state in `tabStateCache` (a `Map<tabId, snapshot>`) so that switching tabs and returning preserves generated summaries and Q&A. When the cached content belongs to a URL that no longer matches the active tab's URL, it's shown as "stale" with a dismissable banner.

### Ollama streaming

`streamChat()` in `sidepanel.js` posts to `/api/chat` with `stream: true` and delegates NDJSON line parsing to `readNDJSONStream()`. Token strings are yielded to `renderStream()`, which appends them to a DOM element and drives the blinking-cursor animation via the `streaming` CSS class.

### `content.js` selector generation

`generateSelector()` walks up the DOM building a path. It stops early at `#id` anchors or semantic landmarks (`article`, `main`, etc.) and strips unstable utility classes. This keeps selectors short and re-usable across sessions.

## Linting & formatting

- **oxlint** — configured in `.oxlintrc.json`; `lib/` is excluded.
- **oxfmt** — configured in `.oxfmtrc.json`; `lib/` is excluded.
- **SonarQube for IDE** — `sonar-project.properties` excludes `lib/**`.

## `lib/` and `icons/`

Both directories are generated artifacts — do not edit by hand.

- `lib/Readability.js` — copied from `node_modules/@mozilla/readability` by `scripts/setup.js`.
- `icons/*.png` — generated by `scripts/setup.js` (pure-Node PNG encoder, no canvas).

To update Readability: `npm update @mozilla/readability && npm run setup`. Commit both `package-lock.json` and `lib/Readability.js`.
